Implementing Preprocessing

You can implement CSS preprocessing in a number of different ways, including several languages and processes to create leaner CSS files. You can implement Less or Sass by creating a compiler and running the compiler from the command-line (server-side) or you can use CodeKit or another tool (client-side) which will compile Less or Sass code that you’ve written and compile it into a CSS file.

Mechanisms used in Less

Implementing Less is very simple when you already know how to write CSS. However, using Less can make your CSS more readable and compact. Firstly, you can define variables in Less in order to reuse them. For example, you can declare the color @color-black as #000, meaning that you don’t need to write the hex code for that color in the future, but rather you can call the variable @color-black. Mixins are another useful feature which allow you to include, or “mix in”, multiple properties from one rule-set into another rule-set, meaning that you do not have to declare classes with the same properties more than once. You can also make use of functions, which enables you to define variables by calculating them within the code (for example, setting something to 50% of a width you already have in the code, or lightening a color by a certain percentage or defining a color based on its orientation to another color on the color wheel). Finally, you can use nested variables, allowing you to stack variables for more efficient inputs for the same output.
